Layout mocks — Site & PI tab

Four directions for better visibility & space management. Each is a clickable, full-page HTML mock styled with the real warm-editorial tokens. Open any one full-screen, compare, and tell me which to build (or mix elements). The previews below are live, scaled-down iframes.

01 · Focus Canvas

Modules collapse to an icon rail; filters live in a slide-over drawer. The map becomes a full-width hero.

  • Maximum map / canvas area
  • Filters one click away, not always on screen
  • Cleanest, most “product” feel
Best when the map is the main workspace and filters are set once.

02 · Top Toolbar

The left filter column is removed; filters become a sticky horizontal toolbar of inline controls and popovers.

  • Reclaims ~20rem width for content
  • Filters read left→right like a query
  • Familiar — closest to today’s layout
Best for fast filter tweaking with wide results tables and map side-by-side.

Open full screen →

03 · Three-Pane Workspace

IDE-style: icon rail · filter accordion · center map · persistent right inspector that updates in place.

  • Scorecard / HCP profile always visible
  • No scroll-jump when selecting
  • Most powerful for deep analysis
Best for analysts who click site→HCP→profile repeatedly and want context pinned.

Open full screen →

04 · Tabbed Results

Compact filter chip bar + pinned map, then Leaderboard / Scorecard / Diversity / Insights / Shortlist as tabs.

  • Kills the long vertical scroll
  • Smallest change from current build
  • Everything fits one screen
Best lowest-risk upgrade — keeps the current structure, just organizes it.
